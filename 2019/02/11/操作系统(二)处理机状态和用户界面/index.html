    第二章：处理机状态和用户界面

    一.处理机的态

    1.概念：

    中央处理机的工作状态

    2.目的：

    为操作系统建立一个保护环境

    3.类别：

    <1>管态(系统态；特权态)：比核态权限低，无核态时执行核态全部功能

    <2>目态(用户态；常态)：只允许用户程序访问自己的存储区域

    管态 目态
    操作系统的程序执行 用户程序执行
    使用全部命令（包括一组特权指令） 禁止使用特权指令
    使用全部系统资源(包括整个存储区域) 只允许用户程序访问自己的存储区域

    【注】:特权指令是指能改变机器状态的，修改特殊寄存器的，涉及外部I/O的指令等

    4.CPU状态转换

    <1>目态—管态：只能通过中断(系统调用；故障；错误命令后果；特权命令等；)

    <2>管态—目态：一般用设置PSW(修改程序状态字)实现

    二.操作系统的用户界面

    1.概念：

    接口是操作系统提供给用户与计算机打交道的外部机制

    2.类型：

    <1>程序接口=系统功能服务页面

    用户程序运行过程中使用系统调用来请求操作系统的服务

    <2>命令接口=操作界面

    用户使用这个操作界面来组织工作流程和控制程序的运行

    作业控制语言 键盘命令 图形化用户界面
    处理系统 批处理操作系统 分时操作系统，PC操作系统 分时操作系统，PC操作系统
    概念 一种脱机方式下包括对作业处理和资源请求的命令 为联机用户提供的一种能直接控制和干预程序的操作命令 一种用户交互集成的图文并茂的视窗操作环境

    三.Linux的基本组成

    Linux内核 Linux Shell环境 linux文件系统 linux应用程序
    概念 操作系统核心，独占执行最低层任务，决定系统性能和稳定性，本身不大 使用者通过壳，shell来沟通kernel 文件存储在磁盘等设备上的组织方法 ~
    Linux内核版本r.x.y{
    r:目前发布的内核主版本；
    x：偶数表示稳定，奇数代表开发中；
    y：错误修补次数；}
    Linux发行版本Redhat Ubuntu Fedora    		 查看版本：uname -r

    第三章：并发处理与死锁

    一.顺序程序设计

    1.概念：

    若干程序段严格先后次序执行

    2.条件：

    单道计算机系统中只有一个程序运行，这个程序独占系统全部资源，不受外界影响

    2.特点：

    <1>顺序性==严格按规定顺序

    <2>封闭性==独占资源，执行环境保证执行过程不受外界影响

    <3>确定性==执行结果可再现，与初始条件有关，与执行时间无关(封闭性保证)

    二.并发程序及特点

    1.并发环境：

    一定时间内，多个程序同时处于运行但未结束状态，且次序随机

    2.概念：

    多个程序同时在系统中运行，执行时间上重迭，即使很小，也称并发执行

    3.描述：
    Code
    4.特点：

    <1>无封闭性==若并发执行共享了公共变量或者改变了另一个程序的变量，程序的执行结果依赖于程序初始条件和执行时的相对速度

    <2>无对应性==一个程序可以对应多个计算，多用户共享使用用一个程序，但处理计算的对象却是不同的

    <3>相互制约性==程序并发执行，公共变量直接互相制约，资源共享间接相互制约

    三.进程的基本概念

    1.进程定义：

    一个程序在给定活动空间和初始环境下，在一个处理机上的执行过程。

    【注】进程控制块PCB：描述进程的动态特征。系统利用PCB控制管理进程；PCB是系统感知进程存在的唯一标志。

    2.进程包括：

    <1>进程的执行程序（一个可执行文件）

    <2>进程控制快（数据结构）

    <3>位于队列，处于某种状态，占用某些系统资源

    3.类型：
    系统进程 用户进程
    定义 操作系统核心代码进程，资源管理和控制 执行用户程序的进程
    区别 系统进程被分配一个初始的资源集合，可为它独占，也能最高优先权资格使用 不能直接I/O操作
    区间 核态 用户态
    4.状态：

    <1>活动规律：

    执行—>暂停—->执行

    <2>基本状态：

    运行状态 就绪状态 等待状态
    定义 已获得运行必须的资源，它的程序正处在处理机上执行 进程等待某个事件的发生而暂停执行 进程已获得除CPU外的运行必备资源
    特点 总只有一个进程处于着这个状态 即使给他CPU时间也无法执行 一旦得到CPU控制全就可立即运行。多个进程处于这个状态

    <3>状态变迁：

    进程的状态不是固定不变的，随自身推进和外界条件的变换而发生变化

    四.进程控制

    1.概念：

    对系统中全部进程实行有效管理，是处理机管理的部分（另一部分是进程调度）

    2.包括：
    进程创建 进程撤销 进程挂起 进程唤醒
    原语形式 create(name,priority,start-addr)
    name：被创建进程的标识符
    priority：进程优先级
    start-addr：某程序的开始地址    		 kill(或者exit)
    无参数，结果无返回值    		 susp(chan)
    chan:进程等待的原因    		 wakeup(chan)
    chan:进程等待原因
    原语功能 创建一个指定标识符的进程，形成该进程的PCB
    UNIX：fork()    		 撤销当前运行的进程归还占用的资源给父进程或操作系统
    从总链摘除PCB结构归还给PCB池
    转入进程调度
    UNIX：exit()    		 终止调用进程的执行，加入等待chan的等待对列，转入进程调度
    UNIX:sleep(chan,pri)    		 当进程等待的事件发生时，唤醒等待该事件的所有进程或等待该事件的首进程
    实现步骤 1.从PCB池申请一个不重复的空的PCB结构
    2.赋予一个进程标识符
    3.初始化PCB相关项
    4.将PCB入就绪队列和PCB总链    		 1.根据撤销进程标识号，从相应队列找到PCB
    2.若该进程拥有子进程，先撤销他的所有子孙进程以防脱离控制
    3.归还拥有的资源给父进程或者操作系统
    4.撤销的进程出队，归还PCB给PCB池
    5.转入进程调度程序    		 1.算法susp(chan)
    2.输入：chan等待原因 输出：无
    {
    进程现场信息->PCB;
    进程状态置为“阻塞”；
    插入到等待“chan”等待序列；
    调用进控制程序；
    }    		 1.从等待进程队列取出PCB
    2.修改PCB信息
    3.修改后的PCB加入有关就绪进程队列
    【注】进程唤醒操作会引起就绪队列和等待chan事件的等待队列发生变化

    五.进程互斥

    1.概念：

    在操作系统中，某一进程正在访问某一存储区域时，不允许其他进程来读出或者修改存储区的内容，这种进程间相互制约的关系即为互斥

    2.互斥原则：

    <1>保证进入

    当有若干个进程欲进入临界区域时，应在有限的时间内使其进入

    <2>排他性

    每次至多有一个进程处于临界区

    <3>有限性

    进程在临界区内仅逗留有限的时间

    {实现基础：锁机制和信号灯机制}

    3.锁和上锁开锁操作：

    <2>锁位：

    <3>问题

    六.进程同步

    1.同步概念：

    并发进程在一些关键点上可能需要相互等待与互通消息，来源于多个进程合作

    互斥：来源于进程对独占使用资源(设备)的竞争

    2.分类：

    <1>诸进程合作完成某工作的逻辑顺序

    <2>对缓冲区资源的共享和传递

    七.死锁

    1.概念：

    多个进程因为竞争共享资源而造成的一种僵局，若无外力作用，这些进程将永远不能再向前推进

    2.产生原因：

    <1>竞争系统资源：系统打印机读卡机都只有1台，可供继承P1P2共享，形成环路。

    <2>进程的推进顺序不当

    3.必要条件：

    <1>互斥条件(资源独占)

    <2>不剥夺条件(不可强占)

    <3>请求和保持条件(部分分配，占有申请)

    <4>环路等待条件

    4.结论：

    <1>参与死锁的进程最少2个

    <2>参与死锁的进程至少有俩个已经占有资源

    <3>参与死锁的所有进程都在等待资源

    <4>参与死锁的进程是当前系统中所有进程的子集

    <5>如果死锁发生，会浪费大量系统资源，甚至导致系统崩溃

    5.解决死锁的基本办法：

    <1>预防死锁

    有序资源分配法：系统给每类资源赋予一个编号，每一个进程按照编号递增的顺序请求资源，释放则相反。

    优点：不会死锁，与资源一次性申请相比提高了利用率

    缺点：存在一定的资源浪费，过于保守，降低计算机效率

    <2>避免死锁

    在系统运行过程中，对进程的资源申请进行动态检查，并根据检查结果决定是否分配资源，若分配后可能发生死锁，则不分配

    系统在进行资源分配之前预先计算资源分配的安全性，若此次分配不会导致系统进入不安全状态，则将资源分配给进程；否则进程等待

    <3>安全状态检测

    如果系统中存在一个安全序列(某种进程执行顺序为每个进程分配所需资源直到最大需求，使每个进程顺序完成)

    实质：枚举出所有进程执行顺序，看能找到一个所有进程最终能顺利完成的方式

    判定：进程序列{P1,…,Pn},对每一个进程Pi，它以后需要的资源量不超过系统当前剩余资源量与所有进程Pj(j<i)当前占有资源量之和

    银行家算法：

    当一个用户对资金的最大需求量不超过银行家现有的资金时，就可以接纳该用户

    用户可以分期贷款，但贷款总数不能超过最大需求量

    当银行家现有的资金不能满足用户的尚需贷款数时，可以推迟支付，但总能使用户在有限的时间里得到贷款

    当用户得到所需的全部资金后，一定能在有限的时间里归还所有的资金

    八.进程通信和线程的基本概念

    1.进程通信概念：

    P.V操作实现进程间的低级通讯，只能传递简单的信号，为低级通讯原语。

    IPC能传递交换大量信息，是多个进程之间相互通信交换信息

    2.Linux进程通信方式：

    <1>管道pipe和有名管道FIFO

    <2>消息队列

    <3>共享内存

    <4>套接字socker(TCP/UDP)

    3.进程并发执行的基础属性：

    <1>资源拥有者：给每个进程分配虚拟地址空间，保存进程映像，控制一些资源(文件I/O设备)，有状态，优先级，调度

    <2>调度单位：进程是一个执行轨迹

    4.线程的定义：

    轻量级进程，进程中的一个运行实体，是一个CPU调度单位，为了简化进程间的通信，以小开销提高进程内并发程度(资源的拥有者还是进程或任务)

    5.线程特点：

    <1>是进程的一个实体，可以作为系统独立调度的基本单位

    <2>不拥有系统资源(只拥有从属进程的全部资源，资源是分配给进程)

    <3>一个进程可以包括多个线程，可并发执行(进程可创建线程执行同一程序的不同部分)

    <4>系统开销小，切换快(进程的多个线程都在进程的地址空间活动)
