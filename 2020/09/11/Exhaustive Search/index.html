Exhaustive Search

ALDS1_5_A:穷举搜索

编写读取序列的程序A的n元素和整数M，和产出“是“如果你能M通过在A，否则“不“.只能使用一次元素。

你得到了序列A和q每个问题包含的问题Mi.

输入

在第一行n是给你的。在第二行，n给出整数。第三行q是给你的。然后，在第四行，q整数(Mi)都给出了。

输出量

每个问题Mi，打印是或不.

约束

  • N≤20
  • Q≤200
  • 1≤元素在≤2000中的应用
  • 1≤Mi≤2000

注记

你可以用BurteForce方法来解决这个问题。假设求解(p，t)是一个函数，它通过在p-元素(包含)之后选择元素来检查是否可以生成t。然后，您可以递归地调用以下函数：

解(0，M)
解(1，M-{1元素之前的元素创建的和)
解(2，M-{从第二元素之前的元素创建的和)
…

递归函数有两个选择：选择p-th元素而不是.因此，你可以检查解题(p+1，t-A[p])和解题(p+1，t)中的所有组合。

例如，下图显示A[0]+A[2]可以生成8。
