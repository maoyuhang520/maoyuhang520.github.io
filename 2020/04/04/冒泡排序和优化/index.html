一般冒泡排序+三种优化

1.一般冒泡排序

c

缺点： 假设数组a[]={1,2,3,4,5,6,8,7} ,只需要第一趟排序交换7，8就可，即上面一般方法无法判断剩下8趟排序是多余的

2.优化1—设flag判断

每排序完一次就判断一次是否有序。有序，return ；无序，进入内层交换。

c

缺点：对于数组a[]={4,3,2,1,5,6,7,8} 这种后半有序前半无序优化不大

3.优化2—减少内部循环比较次数

记下最后一次交换的位置

若后边没有交换，必然是有序的

于是下一次排序从第一个比较到上次记录的位置结束即可

c

4.优化3—鸡尾酒排序

正向扫描找到最大值交换到最后，反向扫描找到最小值交换到最前面

c
